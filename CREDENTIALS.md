# Учетные данные для входа в систему

## Роли пользователей

Система буккроссинга поддерживает три роли:

- **Гость** — роль по умолчанию, не требует входа. Может просматривать каталог книг и список читателей.
- **Читатель (Участник)** — вход по номеру читательского билета (без пароля). Может брать книги и оформлять возврат.
- **Администратор** — вход по паролю. Заводит книги, жанры и читательские билеты, обрабатывает заявки.

## Вход администратора

| Роль | Пароль |
|------|--------|
| Администратор (admin) | `admin2026` |

## Вход читателя (по читательскому билету)

Регистрация читателей вынесена за пределы системы: читательские билеты выдаёт администратор
(раздел «Администрирование» → вкладка «Читатели»). Читатель входит, указав номер своего билета —
пароль не требуется.

Демонстрационные читательские билеты (заводятся администратором, примеры для защиты):

| Читатель | Номер билета |
|----------|--------------|
| Демо-читатель 1 | `BC-001` |

> Важно: номера билетов начинают работать только после того, как соответствующие читатели
> созданы администратором в разделе «Администрирование». До этого в базе есть только созданный мною BC-001.
